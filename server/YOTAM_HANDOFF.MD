## How to run the server

Install dependencies first (one time only):

    cd server
    python -m venv .venv
    source .venv/bin/activate
    pip install flask pandas numpy scikit-learn xgboost pyarrow

**Live mode** (Android app sends real readings):

    python server.py --mode live

**Replay mode** (demo day — no car needed):

    python server.py --mode replay --file data/logs/driver_G_converted.csv --speed 1.0

The server runs on http://0.0.0.0:5000 and is reachable from the
Android phone over WiFi at http://your-laptop-ip:5000.

## The JSON contract

### What the Android app sends (POST /score, once per second)

    {
      "ts": "2026-05-18T12:00:00.000",
      "session_id": "abc123",
      "obd": {
        "speed":        45.0,
        "rpm":          2100.0,
        "throttle":     18.5,
        "engine_load":  34.0,
        "maf":          8.2
      },
      "imu": {
        "accel_x":  0.12,
        "accel_y": -0.04,
        "accel_z":  9.81,
        "gyro_x":   0.02,
        "gyro_y":  -0.01,
        "gyro_z":   0.15
      },
      "gps": {
        "lat":       32.174,
        "lon":       34.838,
        "speed_gps": 44.8,
        "accuracy":  5.0
      }
    }

Fields can be null if the sensor didn't respond that cycle.
session_id is generated by the Android app when a trip starts
and stays the same for the entire trip.

### What the server returns

    {
      "alert":          "YELLOW",
      "score":          0.327,
      "s_traj":         0.58,
      "s_loc":          0.0,
      "s_cont":         0.35,
      "windows_scored": 12,
      "reason":         "Driving pattern is unusual — monitoring"
    }

- alert: GREEN / YELLOW / ORANGE / RED
- score: fused score I in [0.0, 1.0]
- s_traj: behavioral anomaly score [0.0, 1.0]
- s_loc: GPS integrity score [0.0, 1.0]
- s_cont: contextual risk score [0.0, 1.0] — this is your module
- windows_scored: how many 30s windows have been scored this trip
- reason: plain English string for the Android UI to display

## Your integration point — S_cont

Open scorer.py and find this line near the top:

    S_CONT_MOCK = 0.35

And this line inside TripScorer.score_window():

    s_cont = S_CONT_MOCK

Replace the mock with a call to your context risk module. Your module
receives the latest raw reading (the full nested JSON dict) and should
return a float between 0.0 and 1.0.

The integration will look something like this:

    # Replace this:
    s_cont = S_CONT_MOCK

    # With this (adjust import and function name to match your module):
    from context_risk import compute_s_cont
    s_cont = compute_s_cont(latest_reading)

### What your module receives

Your compute_s_cont() function receives latest_reading — the same
nested JSON dict the Android app sends. The fields you care about are:

    latest_reading["gps"]["lat"]     # current latitude
    latest_reading["gps"]["lon"]     # current longitude
    latest_reading["ts"]             # ISO 8601 timestamp
    latest_reading["session_id"]     # trip identifier

For BLE/WiFi fingerprint and paired device check, those signals need
to be added to the JSON contract. Suggested addition:

    {
      "context": {
        "ble_devices":   ["AA:BB:CC:DD:EE:01", "FF:GG:HH:II:JJ:02"],
        "wifi_ssids":    ["HomeNetwork", "CarBluetooth"],
        "paired_obd_id": "OBDLink-MX-001"
      }
    }

Coordinate with Roman so the Android app sends these fields.

## The seven S_cont signals

| Signal           | Trigger                                      | Increment |
|------------------|----------------------------------------------|-----------|
| Time of day      | 03:00–06:00                                  | +0.20     |
| Time of day      | 21:00–01:00                                  | +0.10     |
| Home radius      | Start GPS > 2km from registered base         | +0.20     |
| BLE fingerprint  | Jaccard similarity < 0.2 vs historical       | +0.15     |
| WiFi fingerprint | Analogous to BLE                             | +0.15     |
| Paired device    | Expected OBD/phone not present               | +0.10     |
| Hour/day pattern | Bucket never seen in family history          | +0.10     |
| GPS heatmap      | Departure from never-visited 0.5x0.5km cell  | +0.15     |

Scores accumulate and are capped at 1.0.

---

## Fusion weights

    I = 0.40 x S_traj + 0.33 x S_loc + 0.27 x S_cont

Alert thresholds:

- GREEN:  I < 0.30
- YELLOW: 0.30 <= I < 0.50
- ORANGE: 0.50 <= I < 0.70
- RED:    I >= 0.70
